# 🎬 비디오 편집기

브라우저 기반 비디오 편집 도구 - 역재생, 캡처, GIF 생성

## 🚨 중요: 로컬 서버 실행 필요

이 애플리케이션은 FFmpeg.wasm과 Web Workers를 사용하기 때문에 **반드시 로컬 HTTP 서버에서 실행**해야 합니다.
`file://` 프로토콜로 직접 열면 CORS 에러가 발생합니다.

## 🚀 실행 방법 (가장 간단!)

### ✅ 권장: Node.js 서버 (FFmpeg 지원)

**한 번만 설치:**
```bash
npm install
```

**실행:**
```bash
npm start
```

자동으로 브라우저가 열립니다! (`http://localhost:8000`)

### 대체 방법: Python 서버

**start_server.py 실행:**
```bash
python start_server.py
```

⚠️ 주의: `python -m http.server`는 FFmpeg에 필요한 보안 헤더를 제공하지 않아 역재생 기능이 작동하지 않습니다.

## ✨ 주요 기능

### 1. 📁 비디오 불러오기
- **로컬 파일 업로드**: 컴퓨터에서 직접 비디오 파일 선택
- **URL 입력**: 온라인 비디오 URL로 로드

### 2. 📸 프레임 캡처
- **PNG 캡처**: 현재 프레임을 PNG 이미지로 저장
- **JPG 캡처**: 현재 프레임을 JPG 이미지로 저장 (높은 압축률)

### 3. 🎞️ GIF 생성
- 시작 시간 설정 가능
- 지속 시간 조절 (0.5~10초)
- 프레임 레이트 선택 (5~30 fps)
- 출력 크기 조정 (240px~1920px)
- 실시간 진행률 표시

### 4. ⏪ 역재생 비디오
- 전체 비디오를 역재생으로 변환
- Canvas + MediaRecorder API 사용 (브라우저 네이티브)
- **FPS 선택 가능**: 30fps, 40fps, 50fps, 60fps (기본값: 60fps)
- **품질 선택 가능**: 3~10 Mbps (기본값: 8 Mbps 고품질)
- WebM 형식으로 다운로드 (VP9/VP8 코덱)
- 진행률 실시간 표시
- 로컬 파일 및 URL 모두 지원

## 💡 사용 팁

1. **역재생 품질 설정**:
   - 60fps + 고품질(8Mbps): 매우 부드러운 역재생, 파일 크기 큼
   - 30fps + 중간(5Mbps): 빠른 처리, 적당한 품질
   - FPS가 높을수록 부드럽지만 처리 시간 증가
2. **GIF 생성**: FPS와 크기를 낮추면 파일 크기가 작아집니다
3. **처리 시간**: 큰 비디오 파일은 처리 시간이 오래 걸릴 수 있습니다
4. **URL 비디오**: CORS가 허용된 경우에만 캡처/GIF/역재생이 가능합니다

## 🛠️ 기술 스택

- **Canvas API**: 프레임 캡처 및 비디오 처리
- **MediaRecorder API**: 역재생 비디오 녹화
- **GIF.js**: GIF 이미지 생성
- **HTML5 Video**: 비디오 재생 및 조작
- **WebM/VP9**: 역재생 비디오 출력 형식

## 📝 문제 해결

### GIF 생성이 느림
→ FPS를 낮추거나 지속 시간을 짧게 설정해보세요.

### 역재생 비디오가 끊겨 보임
→ 역재생 옵션에서 **60fps + 고품질(8Mbps)**로 설정하세요.
→ 30fps는 빠르지만 덜 부드럽고, 60fps는 느리지만 매우 부드럽습니다.

### 역재생 비디오 생성이 느림
→ 정상입니다. 60fps 기준으로 모든 프레임을 추출하므로 시간이 걸립니다.
→ 빠른 처리를 원하면 30fps + 낮음(3Mbps)으로 설정하세요.

### WebM 파일이 재생되지 않음
→ VLC Player, Chrome, Firefox 등 WebM을 지원하는 플레이어를 사용하세요.
→ MP4로 변환하려면 온라인 변환 도구나 HandBrake를 사용하세요.
